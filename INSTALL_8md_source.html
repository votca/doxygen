36 ## General (Source) Installation Instructions
37 
38 To install the full package:
39 
40  prefix=WHERE/TO/INSTALL/VOTCA
41  version=master # or 'stable' or 'v1.4.1'
42  git clone -b ${version} --recursive https://github.com/votca/votca.git
43  cd votca
44  mkdir build
45  cd build
46  cmake -DBUILD_CSGAPPS=ON -DBUILD_CTP=ON -DBUILD_XTP=ON -DCMAKE_INSTALL_PREFIX=${prefix} ..
47  make -j5
48  make install
49 
50 ### Resolving the 'not found' dependency errors
51 
52 Assuming all the [dependencies](#dependency-installation) have been correctly
53 installed, one or more might still appear 'not found' upon configuring with
54 `cmake` command (see above). In this case you will need to find out the
55 'non-standard' location for each missed dependency (most often a shared or
56 dynamically loaded library, e.g. `libgromacs.so.*`, `libhdf5.so.*` etc).
57 
58 Error messages produced by Cmake usually provide instructive suggestions for
59 resolving dependency issues. In particular, an appropriate extra `-D` flag is
60 necessary to specify the path to a missed package. You will have to rerun the
61 `cmake` command with the relevant flag(s) added. For example, in the case of a
62 locally installed version of Gromacs:
63 
64  cmake -DBUILD_CSGAPPS=ON -DCMAKE_INSTALL_PREFIX=${prefix} -DWITH_GMX=ON -DGROMACS_INCLUDE_DIR=$HOME/gromacs/include -DGROMACS_LIBRARY=$HOME/gromacs/lib/libgromacs.so ..
65 
66 Be careful to use exactly the option suggested in the error message! You can
67 also add `-LH` or `-LAH` options to the `cmake` command in order to see the
68 available options with brief explanations (note that _changing some of the
69 variables may result in more variables being created_; run `man cmake` for more
70 info).
71 
72 _Only for Linux_: For each dependency package not found by Cmake initially, it
73 might be necessary to add the location of its `lib` directory to the environment
74 variable `LD_LIBRARY_PATH`, **before** building and installing VOTCA, i.e.
75 before running any `make` command. For example:
76 
77  export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:$HOME/gromacs/lib:$HOME/anaconda/lib
78 
79 Note that `LD_LIBRARY_PATH` also needs to be set every time when running an
80 executable from the VOTCA installation afterwards (which can be automated via
81 user's login profile, e.g. in .bashrc). Alternatively, CMake has options to
82 _remember_ where libraries came from at link time, which can be enabled by
83 setting `CMAKE_INSTALL_RPATH_USE_LINK_PATH` to `ON`. VOTCA has enabled this
84 option and a couple of other rpath related tweaks when setting
85 `ENABLE_RPATH_INJECT` to `ON`.

87 ### Common CMake Flags
88 
89 * `BUILD_CSGAPPS` - Build the extra csg applications repo (ON/OFF, Default OFF)
90 * `BUILD_XTP` - Build the xtp repo (ON/OFF, Default OFF)
91 * `BUILD_CTP` - Build the ctp repo (ON/OFF, Default OFF)
92 * `CMAKE_INSTALL_PREFIX` - where to install the votca executables (Default is
93  /usr/local/bin)
94 * `ENABLE_TESTING` - compile tests (ON/OFF, Default OFF)
95 
96 ### Other CMake Flags
97 
98 * `BUILD_CSG_MANUAL` - Build csg pdf manual
99 * `BUILD_CTP_MANUAL` - Build ctp pdf manual
100 * `BUILD_XTP_MANUAL` - Build xtp pdf manual
101 * `WITH_GMX` - Build with Gromacs support (ON/OFF, Default ON)
102 * `CMAKE_DISABLE_FIND_PACKAGE_<name>` - Disable using an optional package called
103  `<name>` (ON/OFF)
104 * `CMAKE_DISABLE_FIND_PACKAGE_HDF5` - Disable using the optional package `HDF5`
105  (ON/OFF, Default OFF; relevant only for the `master` branch)

107 ## Legacy (Source) Installation Instructions
108 
109 Check [dependencies](#dependency-installation) first. Do **NOT** download
110 anything yourself, this is done by `build.sh` below.
111 
112 When installing for the first time, run the following commands in bash:
113 
114  prefix=WHERE/TO/INSTALL/VOTCA
115  mkdir -p ${prefix}/src
116  cd ${prefix}/src
117  wget https://raw.githubusercontent.com/votca/buildutil/master/build.sh
118  chmod +x build.sh
119  ./build.sh --prefix ${prefix} --dev tools csg ctp xtp
120 
121 Replace the _WHERE/TO/INSTALL/VOTCA_ with directory where you want to install
122 votca, usually \\${HOME}/votca is good choice.
123 
124 This commands will:
125 
126 * Create a directory, where the sources are to be stored (above `${prefix}/src`)
127 * Go there
128 * Download our build script
129 * Make the build script executable
130 * Run the build script to download all votca modules using git, it uses the
131  development version, as xtp still undergoes extensive development and no
132  releases are available yet.
