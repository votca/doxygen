3 ## Version 1.4-dev
4 * include manual
5 * an extension of the whole workflow from electrons and holes, to singelt and triplet excitons
6 * a fully functional GW-BSE code optimized for molecular properties, including excited state geometry optimizsation
7 * Inclusion of LIBXC to calculate Exchange correlation matrices, allowing interfacing GW-BSE with many quantum mechanical packages
8 * support for ORCA DFT package
9 * framework to use stochastic models to generate larger system
10 * better performance of larger systems
11 * new calculators: egwbse,igwbse,ewald,.....
12 * support for intel mkl library and compilers for better performance
13 * Aperiodic polarisation embedding to calculate classical configuration energies without cutoffs
14 * xtp_update_exciton to update state file to newest format
15 * integration of moo and kmc into xtp for easier installation
16 * kmc_lifetime calculator to simulate exciton movement with lifetimes
17 * partialcharges to extract atomic charges from qm calculation
18 * renaming from ctp to xtp
19 * many bugfixes

23 ## Version 1.3 (released XX.09.15)
24 * new executables: ctp_tools, ctp_dump, ctp_parallel, xtp_testsuite, xtp_update
25 * ctp_tools wraps light-weight tools that assist e.g. in generating the system mapping file
26 * ctp_dump extracts information from the state file to human-readable format
27 * ctp_parallel wraps heavy-duty job-based calculators, allowing for synchronization across processes
28 * ctp_testsuite provides an easy-to-use environment to run selected tests as well as individual calculators
29 * ctp_update updates an existent state file to the current version
30 * new calculators: edft, idft, pdb2map, xqmultipole, ...
31 * edft / idft (provide interfaces to the GAUSSIAN, TURBOMOLE & NWCHEM packages to compute couplings, internal energies and partial charges)
32 * pdb2map (generates a system mapping file from an input coordinate file)
33 * xqmultipole (computes classical configuration energies of charged clusters embedded in a molecular environment)
34 * enhanced usability via the command-line help, tutorial & test-suite
35 * a GUI tutorial assists with the first practical steps in using VOTCA-CTP
36 * an extended and homogenized help system provides short infos on individual calculator options from the command line
37 
38 ## Version 1.0 (released 23.10.11)
39 * parallel evaluation of site energies (Thole model + GDMA) - Tinker no longer required
40 * much clearer input files (and many more checks for input errors)
41 * most of calculators are parallel and can be used on a cluster
42 * bug in zindo/ctp interface fixed
43 * state file now contains the atomistic trajectory, rigid fragments and conjugates segments
44 * support for several MD frames
