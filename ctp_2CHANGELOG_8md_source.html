3 ## Version 1.3 (released XX.09.15)
4 * new executables: ctp_tools, ctp_dump, ctp_parallel, ctp_testsuite, ctp_update
5 * ctp_tools wraps light-weight tools that assist e.g. in generating the system mapping file
6 * ctp_dump extracts information from the state file to human-readable format
7 * ctp_parallel wraps heavy-duty job-based calculators, allowing for synchronization across processes
8 * ctp_testsuite provides an easy-to-use environment to run selected tests as well as individual calculators
9 * ctp_update updates an existent state file to the current version
10 * new calculators: edft, idft, pdb2map, xqmultipole, ...
11 * edft / idft (provide interfaces to the GAUSSIAN, TURBOMOLE & NWCHEM packages to compute couplings, internal energies and partial charges)
12 * pdb2map (generates a system mapping file from an input coordinate file)
13 * xqmultipole (computes classical configuration energies of charged clusters embedded in a molecular environment)
14 * enhanced usability via the command-line help, tutorial & test-suite
15 * a GUI tutorial assists with the first practical steps in using VOTCA-CTP
16 * an extended and homogenized help system provides short infos on individual calculator options from the command line
17 
18 ## Version 1.0 (released 23.10.11)
19 * parallel evaluation of site energies (Thole model + GDMA) - Tinker no longer required
20 * much clearer input files (and many more checks for input errors)
21 * most of calculators are parallel and can be used on a cluster
22 * bug in zindo/ctp interface fixed
23 * state file now contains the atomistic trajectory, rigid fragments and conjugates segments
24 * support for several MD frames
